# BollingerPyramid v1.0 策略说明

## 概述

布林带突破金字塔策略（BollingerPyramid）是一个基于布林带上轨突破的顺势加仓策略。核心思路：价格突破布林带上轨后顺势做多，按固定点数间距逐层加仓形成金字塔，利用保护损锁定浮盈，配合补单机制在价格震荡中保持仓位完整性。

---

## 策略流程

```
等待信号 → BB突破做多 → 逐层加仓 → 保护损/补单管理 → 整体止盈/止损 → 周期结束 → 等待下一信号
```

---

## 一、入场条件

**触发方式**：使用 **已完成K线（bar[1]）** 的收盘价与布林带上轨比较，避免未完成K线的虚假信号。

| 条件 | 说明 |
|------|------|
| 无活跃周期 | 当前无任何持仓周期运行 |
| Close[1] > BB_Upper[1] | 前一根完成K线的收盘价突破布林带上轨 |
| 新K线检测 | 同一根K线不重复触发信号 |
| 点差过滤 | 当前点差 ≤ `MaxSpreadPoints` |

**入场操作**：以当前 Ask 价开多单（Level 0），记录为周期初始价 **P0**。

---

## 二、金字塔加仓机制

### 2.1 层级结构

```
Level 9  ─── P0 + 9 × LevelSpacing
Level 8  ─── P0 + 8 × LevelSpacing
Level 7  ─── P0 + 7 × LevelSpacing
Level 6  ─── P0 + 6 × LevelSpacing
Level 5  ─── P0 + 5 × LevelSpacing
Level 4  ─── P0 + 4 × LevelSpacing
Level 3  ─── P0 + 3 × LevelSpacing
Level 2  ─── P0 + 2 × LevelSpacing
Level 1  ─── P0 + 1 × LevelSpacing
Level 0  ─── P0（初始入场）
```

### 2.2 加仓规则

- **触发条件**：`bid >= P0 + level × LevelSpacing × Point`
- **逐层构建**：第 N 层加仓要求第 N-1 层有持仓（或被打掉等待补单）
- **每 tick 最多加一层**，避免过度操作
- **最大层数**：由 `MaxLevels` 参数控制（1-10）

### 2.3 仓位标识

每笔持仓的注释格式为 `BP_L0` ~ `BP_L9`，用于标识所属层级。

---

## 三、保护损机制

### 3.1 保护损设置

当持仓 ≥ 2 层时，对 **非最新一层** 的所有持仓设置保护性止损：

```
保护损价格 = 入场价 + ProtectionPoints × Point
```

- 保护损目的是 **覆盖手续费后锁定微利**，不是传统止损
- 只在止损价未设置或不正确时修改，避免频繁请求
- 最新一层不设保护损，给予价格波动空间

### 3.2 保护损被打掉后的补单

当某层保护损被触发后，该层标记为 `SLHit`，进入补单等待状态：

- **补单条件**：价格重新回到该层目标价（`P0 + level × LevelSpacing × Point`）
- **每根 K 线最多补一单**，防止震荡反复触发
- 补单后立即更新保护损

---

## 四、整体止盈/止损

所有持仓共享以 **P0** 为基准的整体止盈止损：

| 类型 | 条件 | 说明 |
|------|------|------|
| **止盈** | `bid >= P0 + TakeProfitPoints × Point` | 平掉所有持仓，结束周期 |
| **止损** | `bid <= P0 - StopLossPoints × Point` | 平掉所有持仓，结束周期 |

止盈止损检查具有 **最高优先级**，在加仓和保护损管理之前执行。

---

## 五、风控管理

### 5.1 回撤保护

- 实时追踪账户净值高水位（`accountEquityStart`）
- 当净值从高水位回撤超过 `MaxDrawdownPct%` 时，**强制平掉所有持仓并结束周期**
- 回撤保护触发后重置高水位基准

### 5.2 点差过滤

当前点差 > `MaxSpreadPoints` 时，跳过所有交易操作（包括入场、加仓、补单）。

### 5.3 保证金检查

每笔开仓前检查可用保证金，要求：

```
所需保证金 ≤ 可用保证金 × 95%
```

保证金不足时跳过该笔交易并记录警告日志。

---

## 六、手数管理

### 6.1 固定手数模式（默认）

`UseDynamicLot = false` 时，所有层级使用固定手数 `InitialLot`。

### 6.2 动态手数模式

`UseDynamicLot = true` 时，按账户余额比例计算手数：

```
手数 = 余额 × RiskPercent% ÷ 单手保证金
```

手数经过标准化处理：向下取整到 `VOLUME_STEP`，限制在 `[VOLUME_MIN, VOLUME_MAX]` 范围内。

---

## 七、状态持久化

EA 使用 MetaTrader 5 的 **全局变量（GlobalVariables）** 持久化周期状态，确保终端重启后能恢复运行：

| 全局变量名 | 存储内容 |
|------------|---------|
| `BP_{Symbol}_{Period}_CycleActive` | 周期是否活跃 |
| `BP_{Symbol}_{Period}_CycleP0` | 初始入场价 P0 |
| `BP_{Symbol}_{Period}_MaxLevel` | 最大层级 |
| `BP_{Symbol}_{Period}_Active_L{n}` | 第 n 层是否活跃 |
| `BP_{Symbol}_{Period}_Entry_L{n}` | 第 n 层入场价 |
| `BP_{Symbol}_{Period}_SLHit_L{n}` | 第 n 层是否被打掉 |

**恢复逻辑**：启动时从全局变量加载周期状态和 SL 标记，然后从实际持仓列表重建层级信息。若周期活跃但无持仓且无待补单层级，自动结束周期。

## 八、参数说明

### 交易设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `InitialLot` | 0.01 | 固定初始手数 |
| `UseDynamicLot` | false | 启用动态手数（按余额比例） |
| `RiskPercent` | 1.0 | 动态手数的余额风险百分比 |

### 布林带设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `BB_Period` | 20 | 布林带周期 |
| `BB_Deviation` | 2.0 | 布林带标准差倍数 |
| `BB_Timeframe` | PERIOD_CURRENT | 布林带时间周期 |

### 金字塔设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MaxLevels` | 10 | 最大金字塔层数（1-10） |
| `LevelSpacing` | 100 | 每层间距（点数） |
| `TakeProfitPoints` | 1000 | 整体止盈点数（距 P0） |
| `StopLossPoints` | 1000 | 整体止损点数（距 P0 反向） |
| `ProtectionPoints` | 30 | 保护损缓冲点数（覆盖手续费） |

### 风控设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MaxDrawdownPct` | 30.0 | 最大回撤百分比 |
| `MaxSpreadPoints` | 20 | 最大允许点差（点） |
| `MagicBase` | 53179 | 魔术号基数（自动加品种哈希+周期） |
| `DebugMode` | false | 调试模式（开启后记录所有日志） |

## 九、运行逻辑时序（每 tick）

```
OnTick()
  │
  ├─ 市场活跃检查 ── 否 → 返回
  ├─ 高水位更新
  ├─ 回撤保护检查 ── 触发 → 平仓 + 结束周期 → 返回
  ├─ 点差过滤 ── 超限 → 返回
  ├─ 重建层级状态（从持仓）
  ├─ 检测保护损被打掉的层
  │
  ├─ 周期活跃？
  │   ├─ 是 → 止盈检查 → 止损检查 → （命中则平仓+结束周期）
  │   └─ 否 → 新K线？ → BB突破？ → 开仓 Level 0 → 返回
  │
  └─ 周期活跃管理：
      ├─ ① 补单（SL被打掉后价格回来）
      ├─ ② 加仓（价格达到下一层级）
      ├─ ③ 更新保护损
      └─ ④ 保存状态到全局变量
```

---

## 十、魔术号与多品种支持

魔术号按以下公式生成，确保不同品种/周期不冲突：

```
magicNumber = MagicBase + hash(Symbol) + Period()
```

其中 `hash(Symbol)` 使用 31 倍哈希算法对品种字符串计算。EA 只管理自身魔术号匹配的持仓，支持同终端多品种多周期并行运行。

---

## 十一、日志系统

日志文件路径：`Common/DataFolder/BollingerPyramid_{Symbol}_{Period}.log`

- **正常模式**：仅记录重要事件（突破、加仓、止盈/止损、补单、错误等）
- **调试模式**（`DebugMode = true`）：记录所有操作细节，包括每次保护损设置

---

## 十二、注意事项

1. **仅做多**：当前版本只支持做多方向，不做空
2. **同品种单周期**：同一品种同一周期只运行一个周期，不支持并发周期
3. **周末暂停**：自动跳过周末交易（加密货币品种除外）
4. **保护损不替代止损**：保护损是小幅盈利保护，整体风险由 `StopLossPoints` 控制
5. **补单有限制**：每根 K 线最多补一单，避免频繁交易
6. **层级限制**：最大 10 层，注释格式 `BP_L0` ~ `BP_L9`，层级编号不支持两位数
